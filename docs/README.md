# Go 诊断框架 (ossre/go)

本项目是一个基于 Go 语言构建的、可扩展的 Linux 系统诊断框架。它旨在提供一个统一的入口，通过模块化的插件来检测和报告系统内核、I/O、网络和通用配置等方面潜在的问题。

## 快速开始

### 前提条件

- Go 1.24+

### 构建

您可以直接通过源码构建或使用 `Makefile` 提供的快捷命令。

```bash
# 进入 Go 项目目录
cd shibin-code/ossre/go

# 方法一：直接构建
go build ./cmd/ossre

# 方法二：使用 Makefile
make build
```

构建成功后，会在当前目录生成一个名为 `ossre` 的可执行文件。

## 使用方法

`ossre` 工具通过子命令提供功能。

### 列出可用模块

`list` 命令可以展示所有已注册的诊断插件及其功能描述。

```bash
./ossre list
```

**示例输出**：

```
kernel  内核参数与内核状态诊断（占位实现）
io      磁盘与文件系统 I/O 诊断（占位实现）
net     网络连通性与性能诊断（占位实现）
system  系统通用资源与健康状态诊断（占位实现）
```

### 运行诊断模块

`run` 命令用于执行一个或多个指定的诊断模块。您需要通过 `--module` 标志指定模块名称。

```bash
./ossre run --module=<模块名>
```

**示例**：

```bash
# 运行内核诊断模块
./ossre run --module=kernel
```

**示例输出**：

```
模块 kernel 运行完成。诊断结果条目数: 0
```
> 注意：当前所有模块均为占位实现，因此不会产生实际的诊断结果。

### 查看版本

`version` 命令用于显示当前工具的版本信息。

```bash
./ossre version
```

**示例输出**：

```
ossre 诊断框架版本: 0.1.0
```

### 获取帮助

使用 `help`, `-h` 或 `--help` 可以查看帮助信息。

```bash
./ossre help
```
